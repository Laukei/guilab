Taking data

The I-V Tester main GUI

Taking data is the core function of I-V Tester. It assumes you have access to a Stanford Research Systems SIM900 with SIM928 voltage source and SIM970 voltmeter modules. Optionally, a SIM922 temperature module may be used to automatically include the temperature of your system. For each of the SIM922 amd SIM970 modules, the channels used must be specified. For more information see Settings.

With these modules attached, I-V Tester can automatically record the current-voltage characteristics of a superconducting nanowire single photon detector. At a bare minimum, the program must know the maximum voltage and step size in volts. It will then calculate the number of steps required to scan this range. Other fields are optional or automatically populated. The full range of fields is as follows, with blue indicating the value may be autopopulated and green indicating the value is mandatory to run.

Name Description
User The name of the person running the measurement
At The date and time the measurement was run
Batch The sample batch
Device The specific device identifier
SMA The SMA channel the device was measured on
Manual? Select to manually enter the temperature (for example, SIM922 not present or disconnected)
Temp The temperature the device is at
Max voltage The maximum value, in volts, to step up to - the negative of this is the minimum voltage
Step size The size of the steps, in volts, in which to approach the maximum voltage
Total steps The total number of measurements to make - approximately equal to the number of seconds the measurement will take
Rbias The bias or load resistor's size, in ohms
Shunt? Toggles whether the electrical model includes a shunt in parallel with the device
Rshunt If Shunt? toggled on, the value here is used in the electrical model in the I-V Plotter
Comments Notes on the device

Once acceptable values have been input, the green "Run" button on the toolbar will begin the measurement. If the measurement needs to be aborted, the red "Halt" button (which illuminates when running a measurement) will allow this.

Once the data has been acquired (or during the measurement with partial data), the data may be passed to the I-V Plotter tool by clicking the Plot button in the toolbar. Data will still continue to be taken if the measurement is not finished, though the Plotter will not update with it. For more information see Plotting data.

Data acquired through the I-V Tester is not automatically saved. However, the program will not exit without prompting to save. The data is saved in a text format, storing all of the settings from the program first, followed by the data shown in the preview window in X, Y format.

If, after taking a dataset, further data is to be obtained from a similar setup, the New (sequential) option may be used from the File menu. This preserves the settings used previously, but scraps the acquired data.

Data may be loaded into the program using the Open option from the File menu. The data may then be amended or passed to the Plotter. Finally, the raw data from the acquisition may be copied to the clipboard by pressing the Export button.

Plotting data

The I-V Plotter GUI

The I-V Plotter is designed to allow rapid, publication-quality, customisable plotting of current-voltage characteristics of a device. The Plotter includes a circuit model, and can calculate the current over the device from this. The Plotter loads its defaults from the I-V Tester's Settings.

The Plotter is comprised of a preview pane on the left-hand side and the control pane on the right-hand side. The preview is the image that is saved when the Export button, bottom right, is pressed. The control pane values can be returned to the defaults by pressing Reset. The Plotter may be closed unceremoniously by pressing Close.

Many of the options in the control pane are self-explanatory. Graph title adds a graph title: a title is suggested automatically, but may be customised using the Title box, which accepts some LaTeX between $ signs (for example, Greek letters). Offsets in the supplied and measured voltages may be reset to zero using the Correct X offset and Correct Y offset checkbox. The metadata may be displayed on the plot by checking Verbose graph, and the axes may be manually defined using pairs of min/max values by checking Manual axes. Major gridlines may be added to the plot by checking Axis gridlines.

The circuit model calculated when Supplied voltage vs device current is checked

The internal circuit model is used when the Supplied voltage vs device current box is checked. In this model, the device current is calculated as Idevice = (( Vsource - V ) / Rbias ) - ( V / Rshunt ). Plotter uses Python's float('inf') to simulate the system where there is no Rshunt. This models the behaviour when a superconducting device becomes resistive and the bulk of the current travels through the shunt resistor. This allows rapid and accurate determining of the the critical current of a superconducting device.

Finally, the export settings set the Width and Height of the plot, and DPI the dots-per-inch resolution. The units for the width and height may be converted to centimetres (cm) or pixels (px) if these are more helpful.

All of the control pane values are reset to defaults when the Plotter is closed.

Settings

The Device settings and Plot settings for the I-V Tester are accessed from the Settings menu. They are preserved between sessions, and are saved to disk on close.

Device settings

I-V Tester's device settings

I-V Tester's device settings are absolutely vital and core to the operation of the system. The program will behave in horrible and unexpected ways if these settings are wrong. The addresses specified must be correct, with the exception of the SIM922 module and channel which may be omitted if manual temperature values are entered into the Tester.

A full description of the settings is provided for posterity. Blue indicates an optional value and yellow indicates attention should be paid.

Name Description Default value
SIM900 address VISA address of the SIM900 mainframe ASRL1
SIM928 voltage source module SIM900 slot containing SIM928 voltage source module 2
SIM922 temperature module SIM900 slot containing SIM922 temperature module 1
Sensor channel SIM922 channel from which to set temperature 1
SIM970 voltmeter module SIM900 slot containing SIM970 voltmeter module
